# field-notes

Site documentation, config templates, and notes from field work. Mostly for reference — mine and occasionally the team's.

## What's in here

Varies. Network diagrams, policy drafts, site survey notes, and the occasional packet capture from a session worth keeping. Organized loosely. Not all of it finished. Most of it accurate enough to be useful.

## On the commit history

I commit when I remember to. Some of these files lived in a local folder longer than they should have. Timestamps are plausible; don't treat them as authoritative.
